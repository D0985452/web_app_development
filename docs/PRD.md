# 讀書筆記本 - 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：在資訊爆炸的時代，人們閱讀書籍後缺乏一個專注且易於整理的空間來記錄心得與評分。為了讓愛看書的人有一個專屬的版面沉澱閱讀收穫，並分享給其他書友參考，因此發想「讀書筆記本」系統。
- **目標用戶**：愛看書的人（一般讀者）、網站管理員。
- **核心價值主張**：提供一個簡單直覺的平台，讓讀者能輕鬆紀錄閱讀過的書籍、留下心得評分，並提供方便的搜尋功能，促進書籍心得與資訊的流通。

## 2. 功能需求
以下列出本系統的主要功能與對應的使用者故事 (User Story)：

1. **書籍記錄與建立功能**
   - 作為 **愛看書的人**，我希望 **能記錄我看過的書名等資訊**，以便 **完整管理我的閱讀清單**。
2. **撰寫心得與評分功能**
   - 作為 **愛看書的人**，我希望 **能針對已讀書籍寫下心得並給予評分（如 1~5 星）**，以便 **記錄當下的閱讀感受留作紀念**。
3. **心得搜尋與閱覽功能**
   - 作為 **任何人（訪客）**，我希望 **能搜尋並公開閱覽他人對某本書籍的評分及心得**，以便 **作為選書或購書的參考**。
4. **帳號註冊與登入管理**
   - 作為 **愛看書的人**，我希望 **能註冊專屬帳號並登入系統**，以便 **管理和維護我個人發布的所有讀書筆記**。
5. **後台文章管理功能**
   - 作為 **管理員**，我希望 **能檢視並管理所有的書籍心得筆記**，以便 **刪除違規或不當的內容，維持平台的品質**。

## 3. 非功能需求
- **技術限制**：
  - 前端與頁面渲染：HTML/CSS, Jinja2 模板引擎
  - 後端系統：Python Flask 框架
  - 資料庫：SQLite (適合初期 MVP 與輕量級資料儲存)
- **效能與安全考量**：
  - 首頁與搜尋結果應在短時間內完成載入。
  - 需要防範基本的 XSS 攻擊（確保使用者輸入的心得很安全，Jinja2 預設會逸出處理）。
  - 對於需要登入才能操作的頁面（新增筆記、管理後台），必須實作基礎的 Session 身分驗證機制。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 記錄書名、撰寫心得與評分功能（CRUD 操作：Create, Read）。
  - 站內書籍心得列表檢視。
  - 基本關鍵字搜尋（按書名搜尋）。
- **Should Have (應該有)**：
  - 使用者註冊與登入系統。
  - 修改與刪除自己寫的心得（CRUD 操作：Update, Delete）。
- **Nice to Have (未來可考慮加入)**：
  - 心得按讚與留言功能。
  - 管理員專屬後台儀表板。
  - 依照書籍分類標籤 / 評分高低進行篩選排序。
